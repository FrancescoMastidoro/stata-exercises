## Background
The `nlsw88` dataset contains 1988 survey data on American women aged 34–46, covering wages, employment, education, and demographics from the National Longitudinal Survey. In applied labor economics, raw survey data almost always contains missing values, implausible entries, and variables stored in the wrong format — learning to detect and document these problems before any analysis is essential to producing credible results.

## Your tasks

1. **Identify missing values across all variables** — After loading the dataset, use `misstable summarize` to get a count of missing observations for every variable. Which variables have the most missing values? What share of observations is missing for `wage`?

2. **Locate impossible values** — Use `summarize` on `hours` (usual hours worked per week) and `ttl_exp` (total work experience in years). Check the minimum and maximum values. Do any values seem implausible given what these variables are supposed to measure? Use `list` with an `if` condition to display the specific observations that look suspicious.

3. **Examine a labelled categorical variable's underlying codes** — Use `tabulate race` to see the labelled categories. Then use `numlabel _all, add` followed by `tabulate race` again to reveal the numeric codes behind each label. Finally, check what happens when you run `summarize race` — does treating a categorical variable as numeric produce a meaningful mean?

4. **Cross-check missingness against a grouping variable** — Use `tabulate south` and then `tabulate south, missing` (adding the `missing` option). Compare the two outputs. Now use a `bysort` loop or `tabstat` to check whether the mean of `wage` differs between observations where `union` is missing versus non-missing — does missingness in `union` appear to be random with respect to wages?

5. *(Bonus)* **Flag and count problematic observations** — Generate a new indicator variable called `flag_problem` that equals 1 if an observation has a missing value for `wage` OR has `hours` greater than 168 (the number of hours in a week). Use `tabulate flag_problem` to count how many observations are flagged. Think about whether you would drop these rows or investigate them further before deciding.

- **Hint 1**: For Task 1, `misstable summarize` produces a compact table — but it only lists variables that *have* at least one missing value. If a variable does not appear, it has no missing values. To compute the missing share for `wage` manually, recall that `missing(wage)` returns 1 when the value is missing and 0 otherwise, so `summarize missing(wage)` (or a `count if` approach) can help you get a percentage.
- **Hint 2**: For Task 4, the `missing` option in `tabulate` forces Stata to show a separate row for observations where the tabulated variable itself is missing, which are silently excluded by default. For the `tabstat` comparison, think about using `tabstat wage, by(union) statistics(mean n)` — but consider that `union` has missing values, so you may want to create a helper variable that distinguishes "union missing" from the two observed categories before running it.
