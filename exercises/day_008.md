## Background
The `nlsw88` dataset contains 1988 National Longitudinal Survey data on working women in the United States, covering wages, hours, occupation, and demographic characteristics. Identifying and handling missing values, implausible observations, and incorrectly stored variables is a critical first step in any quantitative research project — acting on dirty data without cleaning it first can silently bias every result that follows.

## Your tasks

1. **Load and audit missingness** — Load the dataset with `sysuse nlsw88, clear` and run `misstable summarize`. How many variables contain missing values? Which variable has the highest number of missing observations? Record these numbers as comments in your do-file.

2. **Count missings manually** — For the variable `wage`, use `count` with an appropriate `if` condition to count how many observations have a missing wage. Then count how many observations have a *non-missing* wage. Do the two numbers add up to the total number of observations reported by `describe`?

3. **Flag and examine missing wage observations** — Generate a new indicator variable called `missing_wage` that equals 1 if `wage` is missing and 0 otherwise. Use `tabulate missing_wage` to confirm the counts match what you found in Task 2. Then use `summarize hours` separately for observations where `missing_wage == 1` and where `missing_wage == 0`. Do women with missing wages tend to work more or fewer hours than those with observed wages?

4. **Detect impossible values** — Use `summarize` with the `detail` option on the variable `hours` (usual hours worked per week). Inspect the minimum and maximum values. Then use `count` with an `if` condition to identify how many observations report more than 80 hours per week — a value that, while not strictly impossible, may warrant scrutiny in a labor economics context. Separately, check whether any observations report zero or negative hours; write a comment explaining whether these values seem plausible.

5. *(Bonus)* **Data types and labelled variables** — Run `describe` and identify which variables are stored as byte, int, float, or string. Pick one variable that has value labels attached (visible in the `describe` output) and run `tabulate` on it to see the labels. Then use `numlabel, add` followed by `tabulate` again on the same variable — what changes in the output, and why might seeing both the numeric code and the label simultaneously be useful when cleaning data?
