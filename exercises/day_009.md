## Background
The `nlsw88` dataset contains 1988 survey data on working women from the National Longitudinal Survey, covering wages, hours, occupation, industry, and demographic characteristics. In real labor economics research, raw survey data almost always contain missing values, out-of-range entries, and variables stored in unexpected formats — learning to detect and document these problems before any analysis is essential for producing credible results.

## Your tasks

1. **Audit missing values across the dataset** — Load the dataset with `sysuse nlsw88, clear`. Use `misstable summarize` to get an overview of missingness across all variables. Identify which variables have at least one missing observation and note how many observations are missing for each. Write a comment in your do-file stating which variable has the largest number of missing values.

2. **Inspect a continuous variable for impossible values** — Run `summarize hours, detail` to examine the full distribution of usual hours worked per week. Identify any values at the extremes (minimum and maximum) that seem implausible for a survey of employed women. Write a comment explaining your reasoning — what is the highest number of hours per week that could realistically be worked, and does the minimum raise any concerns?

3. **Flag and count suspicious observations** — Using what you found in Task 2, generate a new indicator variable called `hours_flag` that equals 1 if `hours` is greater than 80 or less than 1, and equals 0 otherwise. Then use `tabulate hours_flag` to count how many observations are flagged. Finally, use `list hours wage if hours_flag == 1` to inspect those suspicious records — do the wage values for these observations look consistent with genuine extreme workers, or do they raise further doubts?

4. **Examine variable storage types and value labels** — Run `describe` and focus on the "storage type" column. Identify at least two variables stored as `byte` or `int` that are actually categorical concepts (not true numeric quantities). For one of those variables, run `codebook varname` and report in a comment: how many distinct categories exist, how many observations are missing, and whether any category has very few observations that might cause problems in group comparisons.

5. *(Bonus)* **Cross-tabulate missingness with a grouping variable** — Using the variable with the most missing values (from Task 1), investigate whether missingness is random across racial groups. Use `tabulate race` combined with an appropriate `if` condition, or use `tabulate race, summarize(varname)` to assess whether the share of missing observations differs substantially across groups. Write a comment discussing what it would mean for your analysis if missingness were systematically higher in one group — what kind of bias could this introduce?

> **Fresh hint**: Do not try to write `if race == "white"` or any string version of the label — Stata stores `race` as a labeled numeric, meaning the underlying data are integers even though the output displays words. Run `tabulate race` first and look at the leftmost column of the output table: that column shows the numeric codes (1, 2, 3, …) that you must use in your `if` condition. Once you have both group means and the overall mean, a simple percentage gap calculation is `(group_mean - overall_mean) / overall_mean * 100`.
