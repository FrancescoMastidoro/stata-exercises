## Background
The `nlsw88` dataset contains survey data on American women's labor market outcomes in 1988, including wages, hours worked, occupation, and demographic characteristics. Detecting and handling missing values, implausible observations, and incorrectly stored variable types is a critical first step in any empirical project — failing to do so can silently bias estimates or cause commands to behave unexpectedly without any error message.

## Your tasks

1. **Audit missingness across the dataset** — Load the dataset with `sysuse nlsw88, clear`. Run `misstable summarize` to get a concise overview of missing values across all variables. Identify which variable has the highest number of missing observations. Write a comment in your do-file noting the variable name and the count of missing values, and briefly speculate why that variable might be missing more often than others.

2. **Quantify missingness as a share** — For the variable you identified in Task 1, compute the share of observations that are missing as a percentage of the total sample. Do this using `display` and Stata's built-in `_N` macro rather than doing the arithmetic by hand. Write a comment stating whether you consider this share large enough to be a concern for analysis.

3. **Flag and inspect impossible values** — Run `summarize wage hours tenure` with the `detail` option. Check the minimum and maximum of each variable. Define a clear rule for what constitutes an impossible or highly suspect value in at least one of these variables (e.g., negative hours, zero wages), and use `list` with an appropriate `if` condition to display any observations that violate your rule. Write a comment stating how many such observations you found and what you would do with them before running a regression.

4. **Check and handle a type inconsistency** — Run `describe` and examine the storage types of all variables. Identify at least one variable that is stored as a numeric type but represents an inherently categorical concept (occupation, industry, or similar). Run `codebook` on that variable to confirm it has value labels attached. Then use `tabulate` on it and cross-tabulate it against `union` membership to see whether union rates differ meaningfully across categories. Write a comment explaining why treating this variable as continuous in a regression would be problematic.

5. *(Bonus)* **Construct a clean analysis flag** — Create a new binary variable called `clean_obs` equal to 1 if an observation has no missing values in `wage`, `hours`, `tenure`, and `ttl_exp`, and 0 otherwise. Use `tabulate clean_obs` to report how many observations would survive a listwise deletion on these four variables. Then run `summarize wage` separately for `clean_obs == 1` and `clean_obs == 0` and comment on whether mean wages differ noticeably between the two groups — this matters because listwise deletion can introduce selection bias.

- **Hint 1**: For Task 1, `misstable summarize` only reports variables that actually have at least one missing value — variables with no missingness are suppressed from the output. Pay attention to the column labeled "Obs" versus the total number of observations shown when you run `describe`, so you can rank variables by the gap between the two.
- **Hint 2**: For Task 2, in Stata `_N` always holds the total number of observations currently in memory. You can use it inside a `display` expression together with the count of missing values you read from the `misstable` output — think about how division and multiplication by 100 would give you a percentage, and consider using the `r()` stored results from a command like `count if missing(varname)` to avoid typing the number manually.
- **Hint 3**: For Task 3, when you run `summarize` with `detail`, examine the minimum value carefully for each variable — Stata treats a true missing (`.`) as positive infinity for sorting purposes, so it never appears as a minimum, but a miscoded zero or negative number would. Use `list` with an `if` condition and keep the output short by specifying only the relevant variables in your `list` command.
- **Hint 4**: For Task 4, the `describe` output has a column called "value label" — any variable with a non-empty entry there is storing categories as numbers with human-readable labels attached. Running `codebook varname` will show you the mapping. When you cross-tabulate with `union`, consider using the `row` option in `tabulate` so percentages are computed within each category, making group differences easier to read.
